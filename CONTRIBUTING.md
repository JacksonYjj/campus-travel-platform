# 提交规范

为了保持提交历史清晰，建议使用统一的提交格式：

```text
type: 简短说明
```

## 推荐类型

- `feat`: 新增功能
- `fix`: 修复问题
- `docs`: 文档变更
- `style`: 代码格式调整，不影响逻辑
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建、依赖、配置等杂项

## 示例

- `feat: 新增用户登录接口`
- `fix: 修复订单列表分页问题`
- `docs: 补充项目启动说明`
- `chore: 更新前端依赖版本`

## 提交建议

- 每次提交只做一类改动，避免把多个主题混在一起。
- 提交信息要具体，尽量说明“做了什么”，不要只写“update”或“fix bug”。
- 如果修改涉及前后端联动，建议在说明里补一句影响范围。
