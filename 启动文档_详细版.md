# 校园拼车 + 约团平台 启动文档（详细版）

## 1. 文档目的

本文档用于指导你在 Windows 环境下，完成本项目的本地部署与启动，确保前后端、数据库可以联通并正常访问。

项目技术栈：
- 后端：Spring Boot + MyBatis + MySQL
- 前端：Vue 3 + Vite + Element Plus

项目目录：
- backend：后端工程
- frontend：前端工程
- carpool_group_platform_schema.sql：数据库建表脚本（位于桌面）

1. JDK 17

打开文件：

  datasource:
    url: jdbc:mysql://localhost:3300/campus_travel_platform?useUnicode=true&characterEncoding=utf8&serverTimezone=GMT%2B8&useSSL=false
    username: root
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver

### 5.2 进入后端目录

cd C:\Users\dymind\Desktop\campus-travel-platform\backend

### 5.3 拉取依赖并启动

## 9. 常见问题与解决

### 9.1 mvn 命令不存在

原因：Maven 未安装或环境变量未配置。

解决：
1. 安装 Maven
2. 配置 MAVEN_HOME
3. 把 Maven 的 bin 目录加入 Path
4. 重开终端后执行 mvn -v

### 9.2 npm 命令不存在

原因：Node.js 未安装。

解决：安装 Node.js LTS 版本，重开终端后执行 node -v 与 npm -v。

### 9.3 后端启动报数据库连接失败

重点检查：
1. MySQL 是否启动
2. application.yml 中账号密码是否正确
3. 数据库名是否为 campus_travel_platform
4. 3300 端口是否可用

### 9.4 前端调用接口 404 或 500

检查项：
1. 后端是否在 8080 端口正常运行
2. 前端 vite.config.js 的 proxy 是否正确
3. 后端控制器路径是否为 /api/auth、/api/trips

### 9.5 端口冲突

如果 8080 或前端默认端口被占用：

后端改端口：
在 application.yml 增加：

server:
  port: 8081

前端改代理：
在 vite.config.js 的 proxy target 改成 http://localhost:8081

前端改启动端口：
在 vite.config.js 的 server.port 改为 3001

说明：你当前环境中 3000 常被占用，前端自动切换到 3001 属于正常现象。
